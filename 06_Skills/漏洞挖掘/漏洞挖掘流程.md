# 漏洞挖掘流程

## 目标

系统化地进行漏洞挖掘、测试和报告，提升漏洞发现能力和技术水平。

## 渗透测试阶段

### 1. 信息收集（Reconnaissance）
- **被动信息收集**：WHOIS、DNS 查询、搜索引擎、社交媒体
- **主动信息收集**：端口扫描、服务识别、目录枚举
- **工具推荐**：Nmap、Masscan、Fscan、Nuclei、Shodan、FOFA

### 2. 漏洞扫描（Scanning）
- **自动化扫描**：Nessus、AWVS、AppScan、OpenVAS
- **手动测试**：Burp Suite、OWASP ZAP、sqlmap
- **专项扫描**：Nuclei 模板、自定义 PoC

### 3. 漏洞利用（Exploitation）
- **Web 漏洞**：SQL 注入、XSS、文件上传、文件包含、SSRF
- **系统漏洞**：缓冲区溢出、提权漏洞、内核漏洞
- **工具推荐**：Metasploit、Cobalt Strike、sqlmap、Burp Suite

### 4. 后渗透（Post-Exploitation）
- **权限提升**：Linux 提权、Windows 提权
- **横向移动**：内网渗透、域渗透
- **持久化**：后门植入、权限维持
- **工具推荐**：Mimikatz、Empire、Covenant、Sliver

### 5. 报告撰写（Reporting）
- **漏洞描述**：清晰准确的漏洞说明
- **复现步骤**：详细的操作步骤
- **修复建议**：可行的修复方案
- **工具推荐**：模板化报告、自动化报告生成

## 常见漏洞类型

### OWASP Top 10 (2021)
1. **A01:Broken Access Control** - 失效的访问控制
2. **A02:Cryptographic Failures** - 加密机制失效
3. **A03:Injection** - 注入（SQL、NoSQL、OS、LDAP）
4. **A04:Insecure Design** - 不安全设计
5. **A05:Security Misconfiguration** - 安全配置错误
6. **A06:Vulnerable and Outdated Components** - 组件漏洞
7. **A07:Identification and Authentication Failures** - 认证失败
8. **A08:Software and Data Integrity Failures** - 软件和数据完整性失败
9. **A09:Security Logging and Monitoring Failures** - 安全日志监控失败
10. **A10:Server-Side Request Forgery (SSRF)** - 服务端请求伪造

### 其他常见漏洞
- **文件上传**：Webshell、文件类型绕过
- **文件包含**：本地文件包含、远程文件包含
- **SSRF**：内网探测、服务伪造、文件读取
- **XXE**：XML 外部实体注入
- **反序列化**：PHP、Java、Python 反序列化
- **逻辑漏洞**：支付逻辑、权限绕过、竞态条件

## 测试方法论

### 黑盒测试
- 不了解目标系统内部信息
- 模拟外部攻击者视角
- 从信息收集开始逐步深入

### 白盒测试
- 了解目标系统内部信息
- 可以进行代码审计
- 测试更全面、效率更高

### 灰盒测试
- 部分了解目标系统信息
- 结合黑盒和白盒的优点
- 常见于实际渗透测试项目

## 漏洞验证

### 复现步骤
1. 环境准备（靶场搭建、工具配置）
2. 漏洞触发（输入构造、请求发送）
3. 结果确认（漏洞证明、影响评估）
4. 深入分析（原理分析、利用链构建）

### PoC 编写
- 清晰的漏洞描述
- 完整的复现步骤
- 可验证的漏洞证明
- 详细的修复建议
