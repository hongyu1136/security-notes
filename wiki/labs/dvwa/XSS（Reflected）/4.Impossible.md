# Impossible

## 源码 (`impossible.php`)

```php
checkToken( $_REQUEST[ 'user_token' ], $_SESSION[ 'session_token' ], 'index.php' );

$name = htmlspecialchars( $_GET[ 'name' ] );
//       ^^^^^^^^^^^^^^^ 所有 HTML 特殊字符转为实体

$html .= "<pre>Hello {$name}</pre>";
```

## 比 High 多了什么

| 防线 | 效果 |
|------|------|
| `htmlspecialchars` | `<` → `&lt;`、`>` → `&gt;`、`"` → `&quot;`、`'` → `&#039;`、`&` → `&amp;` |
| CSRF token | 防跨站请求伪造 |

## 为什么无法绕过

```
输入: <script>alert(1)</script>
  ↓
htmlspecialchars → &lt;script&gt;alert(1)&lt;/script&gt;
  ↓
浏览器渲染成: <script>alert(1)</script>（纯文字，不执行）
```

所有 HTML 标签被转义成纯文本，浏览器不会解析为 HTML 元素。任何 XSS payload 都失效。

## 核心教训

**`htmlspecialchars` 是反射型 XSS 的终极防御。** 输出到 HTML 标签内部时，先把用户输入转实体，数据和代码彻底分离。

> ⚠️ 注意：`htmlspecialchars` 只防"内容被当成 HTML 执行"，不等于 XSS 完全防住了——如果用户输入被拼进 `href`、`onclick` 等属性，还需要额外处理。
