## 服务器验证

```php
$token == hash("sha256", hash("sha256", "XX" . strrev("success")) . "ZZ")
```

两步 SHA256：`sha256(sha256("XXsseccus") + "ZZ")`。

## 绕过

Console 手算 token 后提交——和 Low/Medium 一模一样，算法复杂度不影响安全性。

## 三关总结：客户端 vs 服务器校验

| | 客户端校验 | 服务器校验 |
|------|------|------|
| 执行位置 | 浏览器 JS | PHP/后端 |
| 可篡改 | ✅ F12 随便改 | ❌ 只有服务器能验证 |
| 作用 | 提升用户体验（即时反馈） | 真正安全屏障 |
| 这三关的问题 | 服务器期望的 token 基于客户端算出来的值 | 服务器信任了客户端传来的 token |

**教训：token 必须在服务器生成，客户端只负责带着它提交。** 三关算法越来越复杂（MD5 → 反转 → SHA256），CS 学生容易以为"算法复杂 = 安全"——实际上 JS 里的任何运算在 F12 面前都是透明的。
