# Medium

## 源码 (`medium.php`)

```php
$id = $_POST['id'];
$id = mysqli_real_escape_string($GLOBALS["___mysqli_ston"], $id);

$query = "SELECT first_name, last_name FROM users WHERE user_id = $id;";
//                                                          ^^^^ 注意：没有单引号！
```

## 比 Low 多了什么

- 限制 `POST` 方法（GET 参数无效）
- `mysqli_real_escape_string` 转义特殊字符（单引号、双引号、反斜杠等）

## 漏洞在哪

**查询是数字型**：`WHERE user_id = $id` 没有单引号包裹。

`real_escape_string` 只转义**引号类**字符，数字型注入不需要引号。

## 攻击步骤

1. Burp 抓 POST 请求，Send to Repeater
2. `1 or 1=1` → 返回所有用户
3. `1 order by 2` → 确认字段数
4. `1 union select user,password from users` → 脱裤

## 为什么能绕过

| 防护 | 为什么没用 |
|------|-----------|
| POST 限制 | Burp Repeater 直接发 POST |
| `real_escape_string` | 转义引号对数字型注入无效——payload 里根本没有引号 |
| 下拉选择 | 前端控制，Burp 改 `id` 值即可 |

## 关键教训

**`real_escape_string` 不是防 SQL 注入的银弹**——它只处理引号。数字型查询必须额外用 `is_numeric` 或 `intval` 校验。
