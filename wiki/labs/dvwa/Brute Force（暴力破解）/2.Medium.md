# Medium

## 源码 (`medium.php`)

```php
$user = mysqli_real_escape_string($GLOBALS["___mysqli_ston"], $_GET['username']);
$pass = mysqli_real_escape_string($GLOBALS["___mysqli_ston"], $_GET['password']);
$pass = md5($pass);

$query = "SELECT * FROM `users` WHERE user = '$user' AND password = '$pass';";
$result = mysqli_query($GLOBALS["___mysqli_ston"], $query);

// Login failed
sleep(2);
$html .= "<pre><br />Username and/or password incorrect.</pre>";
```

## 比 Low 多了什么

- `mysqli_real_escape_string` 转义引号 — 堵了 SQL 注入
- 失败后 `sleep(2)` 延时 2 秒

## 攻击步骤

1. Burp 抓包，Send to Intruder
2. Positions：`password` 参数加 `§`
3. Resource Pool：Max concurrent requests 改为 **1**（对抗 sleep）
4. Payloads：加载密码字典
5. Start Attack → 看 Length 或响应内容区分成功/失败

## 为什么能绕过

`real_escape_string` 只管 SQL 注入，对暴力破解零作用。`sleep(2)` 只拖慢速度，Burp 单线程照样跑完。

## 登录成功标志

- 响应 `Welcome to the password protected area`
- 失败有 `sleep(2)` 延时，成功无延时 — 看 Response Time 也能判断（时间短的可能是正确密码）
