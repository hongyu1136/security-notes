# Impossible

## 源码

```php
checkToken(...);                              // ① CSRF Token 校验

if(is_numeric($id)) {                         // ② 先判断是不是数字
    $id = intval($id);                        // ③ 强转整数——非数字全变 0
    $data = $db->prepare('... WHERE user_id = :id');
    $data->bindParam(':id', $id, PDO::PARAM_INT);  // ④ PDO 参数化 + 类型绑定
}
```

## 和 High 的区别

| | High | Impossible |
|------|------|-----------|
| 输入过滤 | 无 | `is_numeric()` + `intval()` |
| SQL 执行 | 直接拼接字符串 | PDO 预编译 + `bindParam` |
| 其他 | — | CSRF Token |

## 为什么无法注入

`intval()` 把 `1' AND 1=1#` 变成整数 `1`——注入 payload 被物理消灭。PDO 再补一层：就算绕过 `is_numeric`，`bindParam` 也按类型绑定，参数和 SQL 语句物理隔离。
