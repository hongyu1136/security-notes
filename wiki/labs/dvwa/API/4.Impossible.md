# API OAuth 2.0 练习

## 内容

用 Postman 或 Burp 配置 OAuth 2.0 认证流程——access token 过期后用 refresh token 续期。

## 学习目标

自动化 token 管理，跳过手动复制粘贴。

## 关键概念

- Access Token：短期有效（几分钟），发请求时带在 Authorization 头
- Refresh Token：长期有效，Access Token 过期后用它换新的
- 两者都过期 → 重新登录

## 费曼版

Access Token = 酒店房卡（当天有效）。Refresh Token = 前台身份确认（住一周都能续）。房卡过期拿身份证去前台换新卡。身份证也过期就重新登记入住。
