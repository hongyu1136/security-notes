## 防护

PHP 不做任何事，全放前端。关键改动在 index.php：

```php
// 其他级别：$decodeURI = "decodeURI";  → lang 被 URL 解码
// Impossible：$decodeURI = "";         → lang 保持原始编码
```

```javascript
document.write("<option value='" + lang + "'>" + $decodeURI(lang) + "</option>");
//                 value 写 lang 原始值    显示文本为空（不 decode）
```

## 为什么打不过

| 攻击 | 效果 |
|------|------|
| `<img src=x onerror=alert(1)>` | 卡在 `value` 属性字符串里，不渲染 |
| `<script>alert(1)</script>` | 同上 |
| 引号闭合 | `value` 里可以逃逸但显示文本为空，不触发事件 |

URL 编码不自动解码 + value 属性不渲染 HTML = 无注入面。

## 四关对比

| | 绕过什么 | 核心教训 |
|------|------|------|
| Low | 无过滤 | DOM XSS 根本原因：JS 信任 URL |
| Medium | `stripos("<script")` | 黑名单不可靠，一个 `<img>` 绕 |
| High | PHP 白名单 | `#` 不发给服务器，前后端不同视野 |
| Impossible | — | 不 decode 用户输入，隔离 value 属性 |
