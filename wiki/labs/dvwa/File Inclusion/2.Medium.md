# Medium

## 源码

```php
$file = str_replace(array("http://", "https://"), "", $file);  // 删协议头
$file = str_replace(array("../", "..\\"), "", $file);            // 删目录穿越
```

## 漏洞在哪

`str_replace` 只替换一次，不递归。删掉中间的 `../` 后外层自动拼成完整路径。

## payload

```payload
?page=..././..././hackable/flags/fi.php
```

## 为什么能绕过

| 防护                     | 为什么没用                                     |
| ---------------------- | ----------------------------------------- |
| 删 `../`                | 只删一次——`..././` 删掉中间的 `../` 后外层拼回 `../../` |
| 删 `http://` `https://` | 嵌套绕过，不需要协议                                |

## 关键教训

**黑名单只删一次 = 没防。** 真正的防御是白名单（只允许特定文件），不是过滤关键字。
