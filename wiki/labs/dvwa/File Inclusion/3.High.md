# High

## 源码

```php
if( !fnmatch( "file*", $file ) && $file != "include.php" ) {
    echo "ERROR: File not found!";
    exit;
}
```

参数必须 `fnmatch` 匹配 `file*`（以 `file` 开头），或等于 `include.php`。

## 漏洞在哪

`fnmatch("file*")` 只检查**字符串开头**是 `file`，不检查后面是什么。`file://` 协议也符合。

## 为什么能绕过

| 防护 | 为什么没用 |
|------|-----------|
| `fnmatch("file*")` | 只匹配前缀——`file:///` 以 `file` 开头，通过 |
| 只允许 `file1-3.php` 和 `include.php` | 检查不校验文件真实存在——任何以 `file` 开头的字符串都过关 |

## 关键教训

**通配符检查不是白名单。** 真正的防御是 Explicit Allowlist（Impossible 的 `in_array` 硬编码），不是 `fnmatch("file*")`。
