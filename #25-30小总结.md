## a 关的统一规律

```
a 关 = 同过滤 + 改闭合方式 + 关报错
```

| 主关 | a 关闭合 | 影响 |
|------|---------|------|
| 25 `'$id'` | 25a `$id`（无引号） | 不用闭合，UNION 随便写 |
| 26 `'$id'` | 26a `('$id')`（加括号） | `')` 闭合，多一层括号 |
| 27 `'$id'` | 27a `"$id"`（双引号） | `"` 替 `'`，写法照搬 |
| 28 `('$id')` | 28a `('$id')`（不变） | 但过滤只剩一条，反而更简单 |

## 29/30 — HPP（和其他关完全不同）

29、30 没有 preg_replace 过滤。防御机制是一个**双层架构的坑**：

```
WAF 层 → java_implimentation() → 取查询串里第一个 id → 检查纯数字
PHP 层 → $_GET['id']           → 取最后一个 id      → 不做检查直接拼 SQL
```

所以发 `?id=1&id=-1' union select 1,2,3--+`：

- WAF 看到 `id=1` → 纯数字 ✅ 放行
- PHP 拿到 `id=-1' union select...` → 注入成功

这是整个靶场唯一**不需要改 payload 语法**的关——空格、注释、关键字全都不用绕，只要把参数写两遍。

30 关就是 29 的双引号版本，换闭合符号就行。

## 核心技巧速查

| 场景 | 方法 |
|------|------|
| or/and 被过滤 | `\|\|` / `%26%26` 或双写 `oorr` |
| 空格 `\s` 全禁 | `()` 包裹代替 |
| 空格 `[ +]` 只拦空格和+ | `%0a` 换行 |
| select/union 大小写过滤 | 混合大小写 `sEleCt`，避开三种已知 |
| `union\s+select` 词组 | 双写 `seunion selectlect` |
| 注释全禁 + 报错有 | updatexml / extractvalue |
| 注释全禁 + 无报错 | 布尔盲注（26a）或 `;%00`（27/28 环境支持） |
| 无任何过滤但有 WAF | HPP `?id=数字&id=payload` |
